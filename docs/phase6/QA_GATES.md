# Phase 6 Workflow Quality Gates

This document formalizes the quality criteria (gates) required to pass each stage of the Phase 6 \`dev-workflow\`.

## Stage 1: Initial Requirements
- [ ] Requirements document exists (\`evidence\`).
- [ ] At least one approval from a non-author (e.g., Tech Lead or Architect).

## Stage 2: System/Architectural Design
- [ ] Design document exists (\`evidence\`).
- [ ] At least one approval from Architect or Tech Lead.

## Stage 3 & 4: Briefings
- [ ] Mutual alignment confirmed via review comments.

## Stage 5: Test Case Design
- [ ] Test case document exists (\`evidence\`).
- [ ] At least one approval from QA Lead or Tech Lead.

## Stage 6: Development Implementation
- [ ] Feature branch exists and contains code commits.
- [ ] Implementation notes detail what was changed.

## Stage 7: Integration Testing
- [ ] Test report or execution log exists (\`evidence\`).
- [ ] All P0/P1 bugs resolved.
- [ ] Approval from QA Lead required.

## Stage 8: Go-Live Review
- [ ] Final sign-off from all assigned roles.
- [ ] **MANDATORY**: Approval from the assigned **Tech Lead** is required for completion.
